# LifeOS 使用指南

这份文档给普通使用者看，只讲怎么打开、怎么记录、怎么备份。

## 打开项目

在项目目录运行：

```bash
./run.sh
```

浏览器会自动打开 LifeOS。如果没有自动打开，可以在浏览器输入终端里显示的本地地址。

## 页面说明

### 今日

用来快速记录今天做过的事情。

可以直接在大文本框里写一段流水账，然后点击“让 LifeOS 整理”。LifeOS 会调用后端 AI 服务，把内容拆成多条待确认记录，并尽量自动匹配到已有项目和任务。

确认前你可以修改：

- 做了什么
- 花了多少分钟
- 属于哪个资产分类
- 关联哪个项目
- 关联哪个任务

点击确认后，记录会保存到浏览器本地。快速按钮仍然可以直接记录固定事项。

### 项目

用来创建长期项目，例如考试、课程、作品、健身计划。

项目可以关联任务和时间记录，方便之后看投入是否真的落到重要方向上。

### 任务

用来拆解项目里的具体行动。

任务可以创建、完成、删除。完成任务会参与当天复盘统计。

### 复盘

用来生成当天复盘卡。

复盘会统计：

- 今日总投入时间
- 正向资产时间
- 消耗时间
- 已完成任务数量
- 系统给出的次日建议

复盘卡可以导出 PNG。

### 仪表盘

用来看一段时间内的时间投入结构。

可以按今天、近 7 天、近 30 天或全部时间查看，也可以按分类筛选。

### 资产定义

用来管理你的时间分类。

默认分类包括学业资产、英语资产、身体资产、输出资产、生存任务、注意力消耗、关系资产、财务资产。

预设分类不能删除，但可以停用或修改部分属性。新增分类会保存在本地。

### 备份

用来导出、导入或重置数据。

建议定期导出 JSON，尤其是在清理浏览器缓存、换电脑、换浏览器之前。

## 数据保存在哪里

所有数据都保存在当前浏览器的 `localStorage`。

统一 key：

```text
app:review-system:v1
```

这意味着：

- 不同浏览器之间数据不互通
- 换设备后不会自动同步
- 清除浏览器数据会丢失记录
- GitHub Pages 只托管网页，不保存你的复盘数据

## 导入演示数据

开发者可以先运行：

```bash
node scripts/seed_demo.mjs
```

然后在“备份”页面导入：

```text
data/demo-backup.json
```

导入后可以看到一组示例项目、任务、时间记录和复盘数据。

## 重置数据

在 App 的“备份”页面点击重置。

重置前建议先导出 JSON。重置会清空当前浏览器里 LifeOS 的本地数据。
